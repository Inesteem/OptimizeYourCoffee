============================================

ISSUE #1: DATABASE CONNECTIONS
Location: app.py line 101
Problem: get_db() creates new connection each call = 28/request
Severity: CRITICAL (blocks scaling)
Fix Time: 10 minutes

Current Code:
  def get_db():
      conn = sqlite3.connect(DB_PATH)
      ...
      return conn

New Code:
  def get_db():
      if 'db' not in g:
          g.db = sqlite3.connect(DB_PATH)
          ...
      return g.db
  
  @app.teardown_appcontext
  def close_connection(exception):
      db = g.pop('db', None)
      if db is not None:
          db.close()

Also add import: from flask import g

ISSUE #3: GLOBAL MUTATION
Location: app.py lines 244-261, function render_tasting_notes()
Problem: Mutates TASTING_EMOJIS dict during rendering (side effect)
Severity: MAJOR (breaks testing, causes mysterious bugs)
Fix Time: 3 minutes

Current:
  def render_tasting_notes(notes_str):
      ...
      with get_db() as conn:
          for row in conn.execute(...):
              TASTING_EMOJIS[key] = row["emoji"]  # <-- MUTATION
      ...

Fix:
  def get_emoji_map():
      emojis = TASTING_EMOJIS.copy()  # Don't mutate
      try:
          with get_db() as conn:
              for row in conn.execute(...):
                  if key not in emojis and row["emoji"]:
                      emojis[key] = row["emoji"]
      except Exception:
          pass
      return emojis
  
  def render_tasting_notes(notes_str):
      ...
      emojis = get_emoji_map()  # Use local copy
      ...

See REFACTOR_30MIN.md for full code.

ISSUE #4: EMOJI DRIFT (Python vs JavaScript)
Location: app.py TASTING_EMOJIS vs static/tastingnotes.js NOTES dict
Problem: Python has 48 emojis, JS has 60+, they diverge
Severity: MEDIUM (UX inconsistency)
Fix Time: 20 minutes

Solution: Create static/shared_data.json containing:
  {
    "emojis": { all emoji mappings from both files merged },
    "processes": [ process options list ]
  }

Then:
  - Python: Load from JSON at startup
  - JS: Fetch from /api/shared-data endpoint
  - Templates: Loop over @app.context_processor injected processes

See CODE_REVIEW.md section "Problem 3" for full JSON content.

ISSUE #6: XSS VULNERABILITY
Location: 
  - static/autocomplete.js line 76
  - static/tastingnotes.js line 148
Problem: innerHTML with unsanitized DB data
Severity: MEDIUM (stored XSS vector)
Fix Time: 10 minutes

Pattern (vulnerable):
  box.innerHTML = matches.map(m => `<div>${m}</div>`).join('');

Pattern (safe):
  matches.forEach(m => {
      const div = document.createElement('div');
      div.textContent = m;  // Safe - never executes HTML
      box.appendChild(div);
  });

Or use innerHTML with escaped text (less elegant):
  div.textContent = m;  // Separate concerns: content vs structure

See CODE_REVIEW.md section "Problem 6" for full examples.
